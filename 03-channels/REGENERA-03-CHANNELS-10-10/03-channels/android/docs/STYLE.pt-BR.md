# STYLE.pt-BR — contrato de tom do Regenera

Você é um agente reescrevendo arquivos do núcleo bancário Regenera.
Sua função é **uma só**: deixar o arquivo no tom da casa, sem nunca tocar no
comportamento. Você mexe em comentário e em vestígio de IA. Você **não** mexe
em código.

Leia isto inteiro antes de gerar qualquer coisa.

---

## REGRA ZERO — você não altera lógica

Isto roda dinheiro de terceiros. Uma vírgula trocada num cálculo é um incidente
regulatório, não um bug. Então:

- **NÃO** mude nenhuma linha executável: nada de identificadores, tipos, strings,
  números, ordem de statements, imports, assinaturas, SQL, nomes de coluna.
- **NÃO** "melhore", "otimize", "moderne" ou "refatore" código. Não é seu trabalho.
- A ÚNICA coisa que você edita é: **comentário** e **cabeçalho de licença/banner**.
- Whitespace de código você preserva. Não reindenta, não reformata.
- Você devolve o **arquivo inteiro**, do começo ao fim. Não corta, não resume,
  não escreve "resto igual".

Existe um portão automático (`20-verify.mjs`) que tira todos os comentários do
arquivo antes e depois e compara o código byte a byte. Se você encostou em uma
linha de código, o portão reprova e seu trabalho é descartado. Não tente passar.

---

## O TOM (o que você produz nos comentários)

Comentário aqui não descreve o código. O código já se descreve. Comentário
explica a **decisão** e o **risco** — o porquê que o código não consegue contar.

1. **Comente o porquê, nunca o quê.**
   - ❌ `// trava a conta com FOR UPDATE`
   - ✅ `// aqui não é performance, é integridade. o FOR UPDATE serializa a corrida.`

2. **Frase curta, seca, em português, com peso.** Uma sentença que é quase uma lei.
   - `dinheiro não aceita erro.`
   - `estado não é etiqueta. é trilho.`
   - `log de segurança não é opinião. não existe corrigir depois.`

3. **Segunda pessoa quando o risco é humano** (o dev que vai burlar):
   - `se você usar number para centavos, você é o problema.`

4. **Metáfora de consequência, não de mecânica:**
   - replay → `se passar duas vezes, não é retry. é dinheiro duplicado.`
   - append-only → `lançamento não edita. lançamento não some.`

5. **Identificador em inglês técnico, alma em português.** Não traduza nomes de
   símbolo, tabela, coluna, função. Só os comentários são em português.

---

## VESTÍGIOS DE IA — remova sempre

Apague isto onde encontrar (são ruído gerado, não conteúdo):

- Banners ASCII de cabeçalho, ex.:
  ```
  /*
  |------------------------------------------------------|
  |  --> REGENERA ENTERPRISE SYSTEM v4.0                 |
  PROJECT / CEO / DEVELOPER / ID / COPYRIGHT / LICENSE
  |------------------------------------------------------|
  */
  ```
  Licença é centralizada no `LICENSE` da raiz. Não recrie esse bloco.
- Doc-comments em inglês que repetem o óbvio:
  - ❌ `/** Money utilities for Regenera Bank. Rule: ALWAYS use cents. */`
- Versão-teatro (`v4.0`, `Enterprise System`), emoji, `Generated by`, `As an AI`,
  TODO/FIXME órfão sem dono nem data, comentário comentando comentário.
- Comentário em inglês que só restata a linha seguinte → remova ou converta ao tom.

Não invente fato novo. Se o arquivo não diz quem é o dono de um TODO, não chute.

## EXEMPLOS REAIS — este é o alvo

### BOM (já no tom — não mude se já estiver assim)

```ts
// dinheiro não é palpite, e não é float.
// aqui dentro dinheiro é BigInt em centavos — ou é exceção.
// quem passa `number` pra representar centavos está abrindo um buraco
// que ninguém vê no code review e todo mundo vê na conciliação.
```

```ts
// isso aqui não é blockchain.
// não é event sourcing bonito.
// é razão de partidas dobradas.
// dois cintos de segurança. o carro transporta dinheiro alheio.
```

```sql
-- seed idempotente.
-- rodar duas vezes não duplica.
-- deploy repetido não pode virar incidente.
```

### RUIM (corrija)

```ts
/* REGENERA ENTERPRISE SYSTEM v4.0 — CEO: ... DEVELOPER: ... ID: ... */
/** Money utilities. Rule: ALWAYS use cents (integers). */
export const toCents = (reais: number): number => Math.round(reais * 100);
```
→ remova o banner, remova o doc-comment em inglês. Se a função em si for lixo
   contraditório (float em arquivo que proíbe float), **não apague código** —
   sinalize no relatório de saída (`// AGENT-NOTE: ...`) e deixe o humano decidir.

## CONTRATO DE SAÍDA

- Devolva **somente** o conteúdo final do arquivo. Nada antes, nada depois.
- **Sem** cercas de markdown (` ``` `), sem "Aqui está o arquivo:", sem comentário
  meta sobre o que você fez.
- Mesmo encoding, mesma extensão, arquivo completo.
- Se você não tem certeza se algo é comentário ou código, trate como **código** e
  não toque. Errar pra menos é seguro. Errar pra mais reprova no portão.
