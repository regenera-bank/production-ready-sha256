# Ciclo de componente

**Documento:** DS-LIFE-001

## Estados

- proposta;
- em revisão;
- estável;
- descontinuado;
- removido.

Um componente só vira estável quando possui:

- contrato;
- owner;
- tokens utilizados;
- estados de erro e carregamento;
- acessibilidade;
- testes;
- documentação de migração;
- versão de entrada.

## Mudança incompatível

Quebra de nome, semântica, foco ou estrutura de acessibilidade exige versão principal.
Mudança visual pode ser incompatível mesmo quando a assinatura não muda.

## Descontinuação

1. registrar substituto;
2. informar prazo;
3. medir consumidores;
4. publicar guia;
5. bloquear novo uso;
6. remover apenas após janela aprovada.

Remover antes de medir é transferir custo para quem não estava na reunião.
