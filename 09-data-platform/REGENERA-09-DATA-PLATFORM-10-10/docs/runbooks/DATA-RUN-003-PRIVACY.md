# Runbook — acesso indevido ou exposição de dado

**Severidade mínima:** SEV1 para dado restrito  
**Owner:** privacy + security

## Procedimento

1. revogar grant e sessão;
2. preservar logs e consultas;
3. identificar datasets, campos, sujeitos e janela;
4. bloquear exportações;
5. acionar jurídico e resposta a incidentes;
6. registrar contenção, impacto e decisão.

Log não é limpeza.
É prova.
