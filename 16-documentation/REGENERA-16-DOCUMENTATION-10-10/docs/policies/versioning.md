# Política de Versionamento

- `MAJOR`: mudança incompatível de responsabilidade, fluxo ou controle;
- `MINOR`: capacidade nova compatível;
- `PATCH`: correção editorial sem alterar decisão.

O histórico Git é obrigatório, mas não substitui o campo `version`. Documento substituído recebe `SUPERSEDED` e referência ao sucessor.
