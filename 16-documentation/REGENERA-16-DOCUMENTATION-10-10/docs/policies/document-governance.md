# Política de Governança Documental

## Requisitos de documento ativo

- ID único e estável;
- owner responsável;
- reviewers distintos quando houver aprovação;
- versão semântica;
- classificação;
- data de revisão e próxima revisão;
- fonte de verdade declarada;
- links internos válidos;
- afirmações operacionais apoiadas por evidência.

## Alterações materiais

Mudança de arquitetura, controle, contrato, obrigação, runbook ou responsabilidade exige revisão independente e incremento de versão.

## Proibições

- declarar produção, homologação ou conformidade sem evidência;
- usar checklist marcado como prova isolada;
- ocultar documento substituído;
- incluir chave, token, senha ou dado pessoal real;
- manter duas fontes de verdade ativas para o mesmo ID.
